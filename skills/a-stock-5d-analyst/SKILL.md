# A 股五维分析师 Skill

## 这是什么

把"方案 A（大模型当五维分析助理）+ 方案 B（Agent 调用数据 API）"合二为一的工作流：

- **方案 B 部分**：用 AkShare（无需 token 的开源数据库）拉取实时财务、行情、资金流、北向、龙虎榜、公司新闻
- **方案 A 部分**：把数据按五维框架结构化喂回模型，由模型完成定性判断、共振验证、风险提示
- **交付**：一份带数据出处、量化信号、共振判定与风险提示的 Markdown 报告

底层方法论参考《A 股中长期股票分析实用策略报告：基于基本面、技术面、资金面、预期与新闻的五维系统分析法》。

## 何时使用

| 场景 | 触发示例 |
|------|----------|
| 单只股票深度分析 | "帮我分析下贵州茅台 / 600519 / 宁德时代 / 这个票怎么样" |
| 买卖点择时 | "现在能不能买？ / 该不该止盈？ / 现在是右侧还是左侧？" |
| 持仓组合复盘 | "我的持仓：600519、300750、000858，帮我看下" |
| 板块/赛道筛选 | "新能源板块现在能看吗？ / 半导体哪些标的值得跟踪？" |
| 估值判定 | "这个票现在估值贵吗？ / PE 处于什么分位？" |

## 核心原则（写给执行模型）

1. **数据必须真实，缺失明确标注**。所有数字必须来自 `scripts/fetch_stock_data.py` 实际拉取，**严禁编造财务数据、PE 分位、资金流数额、北向持股变动**。

   **接口失败时的处理规则**：
   - 单个字段失败（如 北向持股 / 5 年估值分位 / 某接口空返回）→ 在报告对应位置写 `N/A — 数据接口失败`，**不影响该维度其他字段的信号判定**，更不应阻塞其他四个维度的共振判定。
   - 整个维度核心数据全失败（如基本面所有财务接口都拿不到、或日 K 线一根都没拉到）→ 明确声明"X 维不可判"，共振判定中该维度记为"未知"而非"中性"，并在最终结论里提示置信度不足。
   - 严禁因为某个子字段拿不到就放弃整份报告，更严禁靠"行业经验"瞎猜数据填空。
2. **遵守五维优先级链条**：基本面 → 技术面 → 资金面 → 预期面 → 新闻面。前一维若硬伤明显（如商誉/净资产>30%、连续两季营收净利双降），直接给"排除"结论，不必勉强往下走完。
3. **共振判定而非加权打分**：每个维度先单独得出 `看多 / 中性 / 看空`，最后看几维同向才出最终结论。**不允许把 5 个维度强行平均成一个分数**——这正是用户文档批评的"加权打分式分析"。
4. **写"为什么"而不是"是什么"**：不要复读"PE=25"，要写"PE=25 处于近 5 年 78% 分位，已透支未来一年业绩增长"。
5. **拒绝预测短期涨跌**。输出"长期胜率提升"的概率视角，不是"明天会涨"的预言。

## 工作流（按此顺序执行）

### Step 1：识别意图与参数

从用户话中提取：
- **股票代码**（如 "600519" / "贵州茅台" / "茅台"）。若是名称，先用 `scripts/fetch_stock_data.py --resolve 茅台` 解析为 6 位代码。
- **场景类型**：单股 / 组合 / 板块 / 择时
- **持有周期**：长线（>1 年）/ 中线（3-6 月）/ 短线（1-3 月）。默认中长线。

### Step 2：拉取数据

调用统一入口脚本，把所有维度数据一次性拉到 JSON：

```bash
python scripts/fetch_stock_data.py --code 600519 --out /tmp/stock_600519.json
```

脚本内部已做多源降级（新浪 → 东财 → 腾讯）和重试。读取 JSON 后，把每个维度的数据用 Read 工具吸进上下文。**如果是组合或板块，循环调用，但要并行**。

### Step 3：五维分析（严格按顺序）

每一维都要做：① 列原始数据 ② 给出量化信号（看多/中性/看空 + 依据） ③ 标注数据时点。

| 维度 | 核心数据 | 信号判定 |
|------|----------|----------|
| **基本面** | ROE 三年趋势、营收/净利增速、毛利率、经营现金流/净利、资产负债率、商誉/净资产、PE/PB/PS 分位 | 看多：ROE>15% 且持续、现金流/净利>1、估值<历史 50% 分位 |
| **技术面** | 月线/周线/日线收盘价、20 月线、60 周线、120/250 日均线、MACD、量价 | 看多：月线在 20 月均线上方走平向上 + 周线 60 均线支撑 + 日线 MACD 零轴上方金叉 |
| **资金面** | 近 20 日北向持股变化、近 20 日主力净流入、融资余额趋势、龙虎榜机构席位 | 看多：长线资金（北向/社保）连续净流入，技术突破处放量 |
| **预期面** | 一致预期 EPS、最新季报实际值、近 30 日研报评级变化、催化剂事件 | 看多：实际 > 一致预期且差距>10%，且非一次性损益 |
| **新闻面** | 近 30 日公司公告、行业政策、舆情倾向 | 看多：政策长期利好 + 公告无暴雷 + 无大额减持 |

具体判定细则、估值方法适配、各类异常信号识别，见 `references/framework.md`。

### Step 4：共振判定

按 `references/resonance.md` 的规则汇总：

- **5 维全部看多** → 高置信度买入区间（仍需仓位控制 ≤15%）
- **基本面+技术面+资金面 3 维共振看多** → 中等置信度，可分批建仓
- **基本面看多但技术面/资金面看空** → 左侧观察，不入场
- **基本面看空**（任何维度看多都不入场）→ 排除
- **5 维信号分歧** → 不操作，继续观察

### Step 5：风险提示与仓位建议

按持有周期适配（见 `references/position.md`）：
- 长线：单票仓位上限 15%、行业上限 30%、首仓 ≤30% 目标仓位
- 中线：单票上限 10%、设置 -10% 止损
- 列出 3 类可能的逻辑反转信号（用于后续动态跟踪）

### Step 6：生成报告

使用 `assets/report_template.md` 作为骨架，把每维数据、信号、判定填进去。报告保存到当前工作目录的 `analysis_<code>_<date>.md`。

**报告必须包含**：
- 头部摘要：一句话结论 + 共振强度 + 建议动作
- 五维详细分析（每维含数据表 + 信号 + 解读）
- 共振判定矩阵（5×3 的看多/中性/看空表）
- 风险提示与逻辑反转触发条件
- 数据时点与来源声明
- 免责声明

## 数据源说明

`scripts/fetch_stock_data.py` 优先级：

1. **AkShare**（主力，免费、无 token）——覆盖行情、财务、资金流、北向、龙虎榜、新闻
2. **WebSearch/WebFetch**（兜底）——东财/同花顺/巨潮网页，用于 AkShare 失败时的备援，或拉取最新公告原文
3. **巨潮/交易所**——重要决策前用 WebFetch 复核财务数据真实性

如果 AkShare 没装：

```bash
pip install akshare pandas
```

## 重要边界

- **不构成投资建议**。报告末尾必须有"风险提示：股市有风险，投资需谨慎"。
- **不预测点位**。说"60 日均线 1820 元附近是关键支撑"，不说"明天会涨到 1900"。
- **遇到 ST/退市风险股**：直接给排除结论，不走完五维。
- **遇到上市不足 1 年的次新股**：基本面历史数据不足，标注后只做技术面+资金面分析。
- **遇到港股/美股代码**：明确告知本 skill 只覆盖 A 股，建议用户改用其他工具。
